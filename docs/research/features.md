# Research: Feature Patterns для ZalAssist

**Topic:** features
**Date:** 2026-04-01
**Project:** ZalAssist — PWA ИИ-помощник для клиентов ЭлектроЦентр

---

## Executive Summary

Три ключевые фичи ZalAssist имеют хорошо изученные реализации. Lowe's (Mylow) и Home Depot (Magic Apron) показали, что проектный диалог ("мне нужно повесить полки") работает лучше поиска по спецификациям — конверсия вдвое выше. Для поиска стандарт 2025 — гибридный: fuzzy matching + семантика. Готовые комплекты эффективны если содержимое раскрыто прямо в карточке. Главная ловушка — открытый chatbot без направляющей структуры.

---

## Findings

### Подбор товаров через диалог с ИИ

**Паттерны:**
- Project-first dialog (Lowe's Mylow): пользователь описывает задачу → ИИ уточняет → список товаров. Двойная конверсия против классического поиска.
- Guided wizard (Zoovu): цепочка вопросов с кнопками
- Quick-reply chips: после каждого ответа ИИ — кнопки с продолжениями

**Рекомендуемая структура:**
1. Стартовый prompt: "Опишите задачу"
2. ИИ: уточняет 1-2 параметра (площадь / тип помещения / мощность)
3. ИИ: выдаёт список товаров с артикулами
4. Максимум 2 уточняющих вопроса до первого результата

**B2B vs B2C:** B2B хочет быстро — форму, не диалог. B2C нуждается в объяснениях. Решение: единый чат с quick-reply chips для B2B-режима.

### Поиск по артикулу, названию, описанию

**Гибридный поиск:** Запрос → нормализация + синонимы → fuzzy matching → результаты

**Синонимы электротехники (обязательный минимум):**
- дифавтомат / АВДТ / дифференциальный автомат
- автомат / АВ / автоматический выключатель
- УЗО / устройство защитного отключения
- щиток / щит / распределительный щит
- ИЭК / IEK / EKF / ЭКФ / Schneider / Шнайдер

**Нормализация артикулов:** убрать дефисы/пробелы, привести к нижнему регистру. "ВА47-29 1P 16А" должен находить "ВА47-29-1P-16A".

### Готовые комплекты

**Ключевая находка (Baymard Institute):** содержимое комплекта должно быть видно в карточке, до клика внутрь.

**Формат карточки:** название + описание задачи + 3-4 позиции с артикулами + кнопка "Полный список" + кнопка "Показать менеджеру".

**Параметрическая адаптация:** площадь → коэффициент → количество. Не нужен ИИ — таблица коэффициентов.

## UX-ошибки

1. **Пустой чат без подсказок** — 3 CTA на старте: "Подобрать / Найти / Комплекты"
2. **ИИ галлюцинирует артикулы** — RAG: топ-20 товаров из поиска в system prompt
3. **Zero-result без выхода** — всегда похожее + "Задать вопрос ИИ"
4. **Длинные диалоги** — максимум 2 уточнения до результата
5. **Мобильный контекст** — шум, одна рука: tap target 48px+, крупный текст

---

## Конфликт: B2B приоритет vs B2C фичи

PROJECT.md называет B2B основным, но 3/3 фичи B2C-ориентированы. Рекомендация: B2B как primary path (быстрый точный поиск), B2C как secondary (диалог и комплекты).
