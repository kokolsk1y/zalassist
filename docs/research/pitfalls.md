# Research: Pitfalls для ZalAssist

**Topic:** pitfalls
**Date:** 2026-04-01
**Project:** ZalAssist — PWA ИИ-помощник для клиентов ЭлектроЦентр

---

## Executive Summary

6 категорий рисков, каждая способна похоронить проект. Наиболее критичны: галлюцинации ИИ при устаревшем каталоге, провал QR-воронки без физической мотивации, и scope creep при разработке одним человеком. Проект выживет если: каталог обновляется регулярно, QR сопровождается конкретным оффером, scope зафиксирован письменно.

---

## PITFALL-01: ИИ рекомендует несуществующие товары

LLM может галлюцинировать артикулы, рекомендовать отсутствующий товар, давать неверные характеристики. Реальные прецеденты: Chevrolet бот галлюцинировал условия сделки, Air Canada выплачивала компенсацию по обещаниям бота.

**Митигация:**
- Системный промпт: только товары из переданного каталога
- Артикулы выводить из JSON, не генерировать текстом
- Дисклеймер: "Наличие уточняйте у консультанта"

## PITFALL-02: Устаревший каталог

Ручная выгрузка = лаг между реальностью и данными. Клиент приходит с неверным артикулом — доверие сломано.

**Митигация:**
- Дата обновления каталога в UI
- Не показывать цены в MVP — только наличие/описание
- Договориться о регулярном обновлении ДО запуска

## PITFALL-03: QR-воронка не работает

QR-код без мотивации сканировать. 53% уходят при загрузке >3 сек.

**Митигация:**
- Текст на QR: "Найди кабель / автомат за 30 сек" — конкретика
- QR минимум A5, на уровне глаз
- Первый экран < 2 сек на 4G
- Рядом с QR — 3 строки что можно сделать

## PITFALL-04: OpenRouter — единая точка отказа

Rate limits, смена цен, downtime, снятие модели с поддержки.

**Митигация:**
- Fallback: поиск по JSON без ИИ при недоступности
- Модель в переменной окружения (быстрая замена)
- Ограничить историю диалога 4 сообщениями
- Зарезервировать $3-4 из $10 на демо руководству

## PITFALL-05: Scope creep при solo-разработке

Инженерная ловушка (RAG вместо простого JSON), хотелки руководства, перфекционизм.

**Митигация:**
- Frozen scope: 3 функции, письменно
- Рабочий прототип через 1 неделю
- Не добавлять фичи без согласования

## PITFALL-06: Prompt Injection

OWASP #1 риск для LLM 2025. Клиенты будут пытаться сломать бота. Скриншоты "смешных ответов" = репутационный риск.

**Митигация:**
- Системный промпт: роль ограничена электротехникой ЭлектроЦентр
- Ограничение длины запроса: 500 символов
- Rate limit по IP: 20 запросов/час

## PITFALL-07: PWA на iOS

Safari ограничивает PWA: localStorage очищается через 7 дней, ограничения API.

**Митигация:** тестировать на реальном iPhone, использовать sessionStorage.

## PITFALL-08: Нет метрик с первого дня

Без аналитики невозможно доказать 15% руководству.

**Митигация:**
- GA4 или Plausible до запуска
- QR с UTM-метками
- Еженедельный отчёт для руководства

---

## Recommendations

### Must Do
1. Ограничить системный промпт: только ассортимент, только из каталога
2. Дата обновления каталога в UI, не показывать цены
3. Fallback при недоступности OpenRouter
4. Аналитика до первого запуска
5. Тест на iPhone с Safari
6. Зафиксировать scope письменно

### Should Do
7. Rate limiting по IP (20 req/час)
8. Конкретная формулировка на QR
9. История диалога ≤ 4 сообщений
10. Зарезервировать $3 на демо
